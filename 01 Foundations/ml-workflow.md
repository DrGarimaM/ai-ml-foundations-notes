# Machine Learning Workflow

Machine Learning projects follow a structured process from defining the problem to deploying a trained model.

## 1. Problem Definition

Clearly define the task that the model needs to perform.

Example:
Predict house prices based on area, location, and number of rooms.

## 2. Data Collection

Gather relevant data required to train the model.

Sources include:
- Databases
- Sensors
- Logs
- APIs
- Public datasets

## 3. Data Preparation

Prepare the data for training.

Tasks include:
- Handling missing values
- Data cleaning
- Feature selection
- Encoding categorical variables
- Normalization

## 4. Model Training

Select a machine learning algorithm and train it using the prepared data.

Examples:
- Linear Regression
- Decision Trees
- Random Forest
- Neural Networks

## 5. Model Evaluation

Evaluate model performance using appropriate metrics.

Examples:

Regression:
- Mean Absolute Error
- Mean Squared Error

Classification:
- Accuracy
- Precision
- Recall
- F1 Score

## 6. Deployment

The trained model is integrated into a real application.

Examples:
- Spam detection
- Fraud detection
- Recommendation systems

## Trainer Notes

Explain the workflow using a simple real-world example such as predicting house prices.

Ask learners:
- Why can't we directly train a model on raw data?
- Why do we need separate training and testing datasets?
